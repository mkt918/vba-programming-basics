連番表示（For〜Next）

新しい概念：ループ変数を「行番号」と「値」の両方に使う

ホーム › STEP 1-B 連番表示

状況説明

太郎さんは「n を入力すると 1 から n まで順番に A列に表示する」プログラムを作りました。

しかし実行すると、A列に正しく連番が並びません。
グループで原因を調べて、正しいプログラムに直してみましょう。

プログラムの仕様

  1. ユーザーに「n（何番まで表示するか）」を入力してもらう
  2. A1 〜 A(n) のセルに、1, 2, 3, … n と順番に表示する

例）n = 5 の場合

セル表示値
A11
A22
A33
A44
A55

バグのあるプログラム

※ 薄赤の行にバグが仕込まれています。

VBA (Excel マクロ) Sub DisplaySequence
1Sub DisplaySequence()
2    Dim n As Integer
3    Dim a As Integer
4    n = InputBox("いくつまで表示しますか？")
5    For a = 1 To n
6        Cells(1, 1).Value = a← バグあり
7    Next a
8End Sub

グループワーク

【課題 1】予想してみよう
n = 5 で実行したとき、A列の結果はどうなると思いますか？
グループで予想してから、実際に実行して確かめましょう。
【課題 2】バグを発見しよう
何行目が間違っていますか？なぜそうなるのか、理由を日本語で説明しましょう。
ヒント：Cells(1, 1) は常にどのセルを指すでしょう？
【課題 3】修正しよう
正しいコードに書き直して、Excel で動作を確認しましょう。

解説（確認後に開く）

▶ バグの正解を見る

バグの場所：6行目  Cells(1, 1).Value = a

理由：Cells(1, 1) は常に A1 を指します。
ループのたびに A1 が上書きされ、最後の値だけが残ります。
行番号もループで変化させるために、1 を a に変える必要があります。

正しいコード：

6        Cells(a, 1).Value = a

ポイント：a を「行番号」としても「表示する値」としても同時に使えることがこのステップの核心です。
