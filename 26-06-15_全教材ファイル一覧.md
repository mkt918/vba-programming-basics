```

│  ├─ 【課題2】バグ発見（何が間違っているか）
│  └─ 【課題3】修正（コードを修正）
```

## 🎯 学習目標と対応

### (1) 知識・技能

**プログラムと情報システムの開発について実務に即して体系的・系統的に理解するとともに、関連する技術を身に付ける**

| STEP | 学習内容 |
|---|---|
| 1-A | For〜Next の構造を理解し、ループ変数と固定値の使い分けができる |
| 1-B | ループ変数を行番号および出力値として同時に活用できる |
| 課題① | ループ内で複数の処理を組み合わせ、複数列への同時出力ができる |
| 2-A | if文と Mod演算子を用いて、偶数判定ができる |
| 2-B | 条件式を反転させ、奇数判定など異なる条件を実装できる |
| 課題② | if〜else を用いて、条件に応じた異なる処理を実装できる |

### (2) 思考・判断・表現

**企業活動に有用なプログラムと情報システムの開発に関する課題を発見し、科学的な根拠に基づいて創造的に解決する力を養う**

| スキル | 教材での実践 |
|---|---|
| バグ発見 | 1-A/1-B/2-A/2-B のバグ修正版で問題点を特定 |
| トレース表作成 | 課題①の「処理内容」表でループ内の動きを追跡 |
| 日本語で表現 | 課題①・課題② で日本語手順書きを記述 |
| フローチャート作成 | （STEP 3以降で導入予定） |

### (3) 主体的に取り組む態度

**企業活動を改善する力の向上を目指して自ら学び、主体的かつ協働的に取り組む態度を養う**

| 要素 | 実践方法 |
|---|---|
| グループ協働 | 課題1で予想 → 課題2で共有 → 課題3で修正 |
| 試行錯誤 | コード実行 → 結果確認 → 修正 を繰り返す |
| 知見共有 | グループワークで発見を説明・共有 |
| 実務意識 | シナリオ（太郎さんが作った〜）で実務感を演出 |

STEP 2-A（偶数表示）★★★
  └ if文 + Mod演算子（条件判定が新規）

  └ if〜else + 複数列出力（複合ロジック）
